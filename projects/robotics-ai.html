Robotics Reinforcement Learning Research Project 2024

Robotics AI Navigation System

Autonomous mobile robot navigation in dynamic environments using deep reinforcement learning and ROS2, with sim-to-real transfer to physical hardware.

2024 — Ongoing
ROS2, PyTorch, Gazebo, Python
Shibata Laboratory, Kyutech
View on GitHub

Overview

This research project develops an autonomous navigation system for mobile robots operating in human-shared, dynamic environments. The key challenge is enabling a robot to navigate safely and efficiently around moving obstacles — including humans — without hard-coded rules, using only onboard sensors.

We train a Proximal Policy Optimization (PPO) agent in the Gazebo simulation environment and investigate domain randomization and adaptive reward shaping techniques to achieve successful sim-to-real transfer to a physical TurtleBot3 platform.

Key Features

🫶

Dynamic Obstacle Avoidance

Agent learns to predict and evade both static and moving obstacles using LiDAR and camera observations.

🌍

Simulation Environment

Custom Gazebo environments with randomized obstacle layouts, textures, and pedestrian models for domain randomization.

⚡

ROS2 Integration

Full ROS2 integration with Nav2 stack. Policy runs as a ROS2 node, interfacing directly with hardware drivers.

📈

Reward Engineering

Adaptive reward shaping with curriculum learning — task complexity increases automatically as agent improves.

Methodology

Observation Space

The agent observes a 360° LiDAR scan (360 rays), relative goal position and distance, current velocity, and a 3-step action history. Observations are normalized and stacked over 4 frames.

Action Space

Continuous action space: linear velocity [0, 0.26 m/s] and angular velocity [-1.82, 1.82 rad/s], matching TurtleBot3 Burger hardware limits.

Reward Function

  • +200 on reaching goal (within 0.2m)
  • -200 on collision
  • +2 × (progress toward goal this step)
  • -0.5 per timestep (time penalty to encourage efficiency)
  • -5 if minimum LiDAR distance < 0.2m (proximity penalty)

Sim-to-Real Transfer

Domain randomization applies random variations in floor friction, obstacle sizes, sensor noise, and lighting at each episode reset. During transfer, we also perform a 50-episode fine-tuning run on the physical robot using a smaller learning rate to adapt to hardware-specific dynamics.

Tech Stack

Python 3.10 PyTorch ROS2 Humble Gazebo Stable-Baselines3 gym / gymnasium TurtleBot3 Ubuntu 22.04 NVIDIA Jetson

Results

Evaluated over 100 test episodes in simulation (50 static, 50 dynamic environments).
  • Success rate (simulation): 93.1% in static, 87.4% in dynamic environments
  • Success rate (real robot): 81.2% after sim-to-real fine-tuning
  • Average path length ratio: 1.18× optimal (near-optimal paths)
  • Inference latency: 4ms per step on Jetson Nano (well within control loop)
  • Training time: ~6 hours on single RTX 3090 (5M environment steps)

This work is directly related to ongoing graduate research at Shibata Laboratory. A related publication appeared at IROS 2024.
