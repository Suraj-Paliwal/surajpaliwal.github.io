AI Chatbot with RAG

A domain-specific conversational AI powered by large language models with retrieval-augmented generation for accurate, knowledge-grounded responses.

2024
Python, LangChain, FastAPI
View on GitHub

Overview

This project implements a full-stack AI chatbot system that uses Retrieval-Augmented Generation (RAG) to provide accurate, context-aware answers grounded in a custom knowledge base. Unlike pure LLM approaches, RAG systems can be updated without retraining and substantially reduce hallucination on domain-specific topics.

The system ingests documents (PDFs, web pages, Markdown files), chunks and embeds them into a vector database, and retrieves the most relevant passages at query time before passing them to the LLM as context. The result is a chatbot that gives accurate, sourced answers on your custom domain.

Key Features

📄

Document Ingestion

Supports PDF, Markdown, HTML, and plain text. Automatic chunking with configurable overlap and size.

🔍

Semantic Search

ChromaDB vector store with OpenAI or local embeddings for fast, accurate semantic retrieval.

💬

Multi-Turn Dialogue

Maintains conversation history with a sliding context window for coherent multi-turn interactions.

⚙

REST API

FastAPI backend with async endpoints, streaming responses, and OpenAPI documentation.

🔐

Source Citations

Every response includes source document references and confidence scores for transparency.

🚀

Docker Deployment

Fully containerized with Docker Compose. Single command deployment of the complete stack.

Technical Architecture

The system follows a standard RAG pipeline with several enhancements:

  1. Ingestion Pipeline — Document loading, cleaning, recursive text splitting, and batch embedding generation using sentence transformers.
  2. Vector Store — ChromaDB persists embeddings locally. Collections are namespaced per knowledge domain.
  3. Retrieval — Maximum Marginal Relevance (MMR) retrieval balances relevance and diversity to avoid redundant context.
  4. Generation — A structured prompt template injects retrieved context, conversation history, and a system persona before calling the LLM.
  5. API Layer — FastAPI handles requests, manages sessions, and streams tokens back to the client.

Tech Stack

Python 3.11 LangChain FastAPI ChromaDB OpenAI API Sentence Transformers Docker Pydantic HTTPX

Results

Evaluated on a domain-specific QA benchmark of 500 question-answer pairs extracted from technical documentation:

  • Answer accuracy: 87.4% (vs. 61.2% without RAG)
  • Hallucination rate: 4.1% (vs. 31.7% without RAG)
  • Average latency: 1.2s to first token (streaming)
  • Retrieval precision@5: 0.91
